# 重构路线图

## 当前诊断

- 静态页面通过一个超大的全局样式文件和一个超大的全局脚本强耦合。
- 页面数据、UI 标记、事件处理、持久化逻辑混在同一个 HTML 文件里。
- 开源协作缺少工程元数据、CI、模块边界和贡献规则。

## 迁移顺序

1. 保持所有旧 HTML 页面可运行。
2. 将共享静态数据迁移到 `src/data`。
3. 将页面交互迁移到 `src/js`。
4. 将新样式迁移到 `src/css`。
5. 逐页替换内联事件处理器。
6. 将 `cc-common.js` 拆成小模块。
7. 将 `cc-styles.css` 拆成基础样式、组件样式、页面样式。

## 模块边界

```txt
src/
  css/       可维护的新样式
  data/      工具注册表和静态元数据
  js/        页面控制器和浏览器工具函数
scripts/    本地检查和重构辅助脚本
docs/       架构和贡献文档
```

## 技术债偿还顺序

1. 修复旧页面编码乱码。
2. 清理内联事件处理器。
3. 合并重复的 localStorage 工具函数。
4. 拆分巨大的 `cc-common.js`。
5. 拆分巨大的 `cc-styles.css`。
6. 清理旧版 Todo 等重复页面。
